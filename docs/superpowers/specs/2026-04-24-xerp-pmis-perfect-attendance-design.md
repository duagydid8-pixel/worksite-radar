# XERP & PMIS 만근 통계 설계

날짜: 2026-04-24

## 목표

`XERP & PMIS` 화면에 `만근 통계` 탭을 추가한다. 사용자는 PH4, PH2 현장별로 토요 현장근무일을 직접 등록하고, 선택된 날짜가 속한 월 기준으로 직원별 만근 여부를 확인한다.

## 만근 기준

선택된 날짜의 `YYYY-MM`을 기준 월로 사용한다.

만근 대상 근무일은 다음 날짜다.

- 월요일부터 금요일까지의 평일
- 사용자가 해당 현장에 직접 등록한 토요일 현장근무일
- 일요일은 항상 제외

직원은 대상 근무일 전체에서 아래 조건을 모두 만족해야 만근이다.

- 결근 0일
- 지각 0회
- 대상 근무일마다 `공수합계(A+B)`가 1.0 이상

예외 조건:

- 해당 일자의 `가산사유`에 `예비군`이 포함되어 있으면 `공수합계(A+B)`가 1.0 미만이어도 출근 인정으로 처리한다.
- 예비군 인정일은 만근 요약에서 별도로 집계한다.

조퇴는 별도 퇴근시간 기준으로 판정하지 않는다. 공수 합계가 1.0 미만이면 조퇴, 지각, 기타 사유와 관계없이 만근 탈락 사유인 `공수미달`로 표시한다.

## 화면 구성

`XERP & PMIS`의 뷰 전환 버튼을 다음 3개로 확장한다.

- `일별 현황`
- `월별 통계`
- `만근 통계`

`만근 통계` 탭에는 다음 영역을 배치한다.

- 토요 현장근무일 관리
- 월별 요약 카드
- 만근자 목록
- 탈락자 사유 목록

### 토요 현장근무일 관리

현재 선택된 현장(`PH4` 또는 `PH2`)에만 적용된다.

기능:

- 날짜 입력으로 토요 현장근무일 추가
- 등록된 토요일 목록 표시
- 등록된 토요일 삭제
- 토요일이 아닌 날짜를 추가하려고 하면 오류 toast 표시

저장 구조:

- Firestore `worksite_data` 컬렉션에 현장별 문서로 저장
- 문서 예: `perfect_attendance_saturdays_PH4`, `perfect_attendance_saturdays_PH2`
- 데이터 형태: `{ dates: string[] }`

## 데이터 계산

기존 `dateMap` 데이터를 사용한다.

직원 식별 키:

- `사번`이 있으면 `사번`
- 없으면 `성명`

월별 직원 목록:

- 기준 월의 저장된 날짜 데이터에 등장한 직원을 집계한다.
- 퇴사자 명단에 포함된 직원은 기존 이상근태 계산과 동일하게 제외한다.

출근 인정:

- 해당 날짜의 직원 row를 찾는다.
- `가산사유`에 `예비군`이 포함되어 있으면 출근 인정
- 그렇지 않으면 `공수합계AB`를 숫자로 변환해 1.0 이상이면 출근 인정
- row가 없으면 결근
- row가 있어도 공수 1.0 미만이면 공수미달

지각 판정:

- 기존 `isLateTime` 기준을 재사용한다.
- `xerp출근`이 있으면 우선 사용하고, 없으면 `pmis출근`을 사용한다.
- 07:10 이후 출근이면 지각으로 집계한다.

## 표시 항목

요약 카드:

- 대상 근무일 수
- 만근자 수
- 탈락자 수
- 예비군 인정 건수

만근자 목록:

- 팀명
- 직종
- 사번
- 성명
- 출근 인정일 수 / 대상 근무일 수

탈락자 사유 목록:

- 팀명
- 직종
- 사번
- 성명
- 결근일수
- 지각횟수
- 공수미달일수
- 예비군 인정일수
- 상세 사유

상세 사유는 날짜 단위로 요약한다.

- `04/03 결근`
- `04/08 지각`
- `04/18 공수 0.5`

## 검증

필수 확인:

- `npm run build`
- 만근 통계 탭 진입
- PH4와 PH2의 토요근무일이 서로 분리 저장되는지 확인
- 토요일 등록/삭제가 화면과 Firestore 저장 상태에 반영되는지 확인
- 일요일은 대상 근무일에 포함되지 않는지 확인
- 평일과 등록 토요일에서 공수 1.0 미만이면 탈락하는지 확인
- `가산사유`에 `예비군`이 포함된 행은 공수 1.0 미만이어도 출근 인정되는지 확인
