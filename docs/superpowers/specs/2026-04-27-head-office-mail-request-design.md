# 본사 메일송부 창 설계

## 목표

본사에 증명서 요청 메일을 보낼 때, 사용자가 이름만 입력하면 신규자명단 데이터에서 인적사항을 찾아 증명서 요청 표를 자동 생성한다. 받는사람과 참조는 사용자가 회사 메일 화면에서 직접 입력한다.

## 화면

새 탭 이름은 `본사 메일송부`로 둔다.

화면은 업무 도구 형태로 구성한다.

- 증명서 종류 선택: `재직증명서`, `경력증명서`, `원천징수영수증`
- 요청일: 기본값은 오늘 날짜
- 이름 입력: 한 명씩 추가하거나 줄바꿈/쉼표로 여러 명 추가
- 요청자 목록: 신규자명단에서 찾은 데이터와 누락 상태 표시
- 미리보기 표: 메일 본문에 붙여넣을 표 표시
- 복사 버튼: 제목 복사, 표 복사

## 자동 생성 규칙

메일 제목 형식:

```text
평택 P4 초순수 {증명서종류}요청의 件_{YYYY.MM.DD}
```

표 컬럼:

```text
NO / 성명 / 주민번호 / 주소 / 현장명 / 입사일 / 비고
```

표 데이터는 신규자명단에서 가져온다.

- `성명`: 입력한 이름과 신규자명단 성명 일치
- `주민번호`: 신규자명단에 있는 주민번호 또는 식별번호 컬럼
- `주소`: 신규자명단 주소 컬럼
- `현장명`: 신규자명단 현장명 컬럼이 있으면 사용하고, 없으면 기본값 `사업팀 [삼성전자 평택 P4-Ph4 초순수 현장]`
- `입사일`: 신규자명단 입사일 컬럼
- `비고`: 기본값 빈칸

## 복사 방식

메일 발송은 앱에서 직접 하지 않는다. 사용자는 회사 메일 또는 ERP 메일 화면에서 받는사람과 참조를 직접 입력한다.

앱은 다음 두 가지를 클립보드로 복사한다.

- 제목: 일반 텍스트
- 표: HTML 표와 텍스트 표를 함께 클립보드에 넣어 Outlook/브라우저 메일에 붙여넣기 좋게 한다. 브라우저가 HTML 클립보드를 지원하지 않으면 텍스트 표로 복사한다.

## 데이터 흐름

1. 신규자명단 저장 데이터 또는 현재 로드된 신규자명단 데이터를 불러온다.
2. 사용자가 이름을 입력한다.
3. 이름을 정규화해서 신규자명단에서 매칭한다.
4. 찾은 사람은 표 행으로 변환한다.
5. 찾지 못한 이름은 누락 목록에 표시하고, 사용자가 직접 보정할 수 있게 한다.

## 예외 처리

- 신규자명단 데이터가 없으면 업로드/저장 필요 안내를 표시한다.
- 같은 이름이 여러 명이면 후보를 보여주고 사용자가 선택하도록 한다.
- 필수 정보가 비어 있으면 해당 셀을 빈칸으로 두고 누락 배지를 표시한다.
- 이름을 찾지 못해도 표에서 삭제하거나 직접 입력할 수 있게 한다.

## 테스트

- 증명서 종류별 제목 생성 테스트
- 이름 입력 파싱 테스트
- 신규자명단 행을 증명서 요청 행으로 변환하는 테스트
- HTML/텍스트 표 생성 테스트
